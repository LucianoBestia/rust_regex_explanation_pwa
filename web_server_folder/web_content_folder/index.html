Rust Regex Explanation PWA

Click inside the Regex field (first field) and press the right arrow. You will immediately see the results.

On every key stroke in the Regex field, this PWA will calculate and show you the explanation, result of Regex methods and the Rust code to use Regex.

Regular expressions or Regex is a special programming language for search, match, validate and replace sub-strings.

It looks very cryptic. But it is powerful and you should use it. Look how crazy is this:

^[a-zA-Z0-9.!#$%&’*+/=?^_`{|}~-]+@[a-zA-Z0-9-]+(?:\.[a-zA-Z0-9-]+)*$

NO PANIC ! Try it !

Click inside the Regex field (first field) and press the right arrow. You will immediately see the result.

On every key stroke in the Regex field, this PWA will calculate and show you the explanation, result of Regex methods and the Rust code to use Regex.

Use this web app when you are coding in Rust and need to write a Regex. Enter your Regex, substitution and test_string.

Change your Regex gradually and observe the result. Great way to learn!

The Regex explanation is 100% human readable. Easy like that!

This is still in beta and a few minor bugs will be corrected soon.

Every TextArea element is manually resizable (grab the bottom-right corner) if you need more or less space.

This PWA (progressive web app) runs entirely inside your browser with Rust Wasm/WebAssembly.

The web server only provides the files. No code is executed on the server.

This means you can use this web app offline. No need for internet connection, except for downloading the code the first time.

Regex has many features and libraries are not 100% compatible. This "flavors" have differences that are subtle, but important for practical use.

The Rust code of this app uses the Rust Regex library: https://docs.rs/regex/

If you also use the same Regex library, this PWA will show you exactly how Regex will work in your code.

Exercise

Go to the page https://en.wikipedia.org/wiki/Terminator_(character) and find all the T model names.

Not very amusing for humans. Simplify it. Copy this shorter text into the field test_string:

T-1000 (Robert Patrick) Terminator known as T-101 T-800 that managed to kill John Connor explicitly named T-600s and T-1000. it jams its remaining hydrogen fuel cell into the T-X's mouth from a T-1000 sent to kill her who has been transformed into a T-3000 improvement over the earlier T-600 units also refers to the character as T-850 used the T-800 and T-850 nomenclature memory of a T-888 model, tearing a malfunctioning T-600 in half

---

The names are always like T-something. Try the first regex to match literally: T-

After that are some digits: T-\d+

There are always some exceptions, the T-X model. Add it: T-(X|\d+)

Let replace T- with the word Robot(). In the substitution field write: Robot($1)
